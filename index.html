SmartBudget aligns with UN SDG 1: No Poverty

This course project was built with social purpose at its core. SmartBudget targets financial exclusion — one of the root causes of persistent poverty — by equipping B40 communities in Malaysia with free, accessible tools for budgeting, financial literacy, and community resource discovery.

A216553 · Praavieen Rajj
Universiti Kebangsaan Malaysia
💸

Addressing Financial Exclusion

Millions in Malaysia's B40 group lack access to financial planning tools. SmartBudget delivers envelope budgeting and expense tracking with zero cost barrier — offline-first so it works without reliable internet.

📚

Building Financial Literacy

The app's knowledge feed (powered by Retrofit REST) surfaces curated financial literacy content — teaching users how to save, budget, and avoid debt traps that perpetuate the poverty cycle.

📍

Community Resource Mapping

GPS integration and Firebase Firestore enable a shared map of financial aid centres, food banks, and government assistance programmes — connecting communities to resources they didn't know existed.

🤝

Technology as a Social Equaliser

Every architectural decision — offline-first Room DB, free Firebase tier, responsive Material 3 UI — was made with low-income users in mind. Tech should lift people up, not leave them behind.

Course Reflection

Mobile Application Programming · Instructor: Sir Nelson Sana · A216553 · Praavieen Rajj
01 · What I Learned

From Zero to Production Android

Before this course I had never built a native Android application. Over one semester I went from understanding Compose layouts to implementing a multi-layered app with Room, Retrofit, GPS hardware, and Firebase Firestore. Every lab built on the last — layout → navigation → state → storage → hardware/cloud. That structured progression made the complexity manageable.

02 · Biggest Challenge

Architecting for the Real World

The hardest part wasn't any single technology — it was learning to think architecturally. Making Room, Retrofit, and Firebase coexist cleanly inside an MVVM structure, with coroutines handling all async work, required a mental model I had to rebuild several times. Sir Nelson's emphasis on clean architecture pushed me to think like an engineer, not just a coder.

03 · SDG Connection

Building with Purpose

Anchoring SmartBudget to SDG 1: No Poverty changed how I approach software design. Every feature decision asked: "does this serve someone with limited data, an old device, and no prior financial knowledge?" That constraint made me a better engineer. Purpose-driven development forces you to prioritise ruthlessly and build with empathy.

04 · Growth & Next Steps

The Engineer I'm Becoming

This course bridged my academic learning with my part-time work at JurisTech. Concepts taught in class — state management, async patterns, clean separation of concerns — showed up directly in production code I was writing professionally. Going forward I want to deepen my work on real-time systems and explore how SmartBudget could scale into a full community fintech platform.

"The best technology is invisible — it solves real problems for real people without asking them to understand how it works. SmartBudget taught me that building for impact means building for the person who needs it most, not the engineer who builds it."
